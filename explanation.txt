Task 1: Understanding the Flow

The email form works as follows: When a user visits the /formtest page, they see a form to enter an email and a list of previously saved emails retrieved from the session. Upon submitting the form via POST, the email is validated for format and constraints, then stored in the session array if valid, and the page redirects back to display the updated list with feedback messages. This creates a simple session-based email collection system where data persists across requests without a database.

Questions and Answers:

What's the difference between GET and POST?
GET is used to request data from a server, like loading a page, and the data is sent in the URL. POST is used to send data to the server, like submitting a form, and the data is sent in the request body, so it's not visible in the URL.

Why do we use @csrf in forms?
@csrf adds a hidden token to prevent Cross-Site Request Forgery attacks. It's Laravel's way of protecting forms from being submitted by malicious sites pretending to be the user.

What actually is the session that's used for in this activity?
The session is like temporary storage on the server that remembers data between page requests. In this activity, it's used to store the list of emails so they persist as the user adds or removes them without needing a database.

What happens if the session is cleared?
If the session is cleared, all the stored emails would be lost, and the list would be empty the next time you visit the page. It's like resetting the temporary memory.
